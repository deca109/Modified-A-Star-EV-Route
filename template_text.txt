Final Year Research Project Report Guidelines and Template
After completion of the project work, every group of students will submit one group wise project report and one individual copy, which should contain the following:
[ListParagraph] Cover Page/ Title Page
[ListParagraph] Certificate by the Guide
[ListParagraph] Acknowledgment (The candidate may thank all those who helped in the execution of the project.) 
[ListParagraph] Declaration of students
[ListParagraph] Report Approval 
[ListParagraph] Preface (Abstract) (It should be in one page and include the purpose of the study; the methodology used and a summary of the major findings.)
[ListParagraph] Table of Contents 
[ListParagraph] Detailed description of the project (This should be split in various chapters/sections with each chapter/section describing a project activity in totality). This portion of report should contain all relevant diagrams, tables, flow charts, software programme, print outs, photographs etc., which are properly labeled.
[ListParagraph] Conclusion
[ListParagraph] Appendices
[ListParagraph] Appendices are provided to give supplementary information, which if included in the main text may serve as a distraction and cloud the central theme.                                    
[ListParagraph] Appendices should be numbered using Arabic numerals, e.g. Appendix 1, Appendix 2.
[ListParagraph] Appendices shall carry the title of the work reported and the same title shall be listed in the Contents page also.
[ListParagraph] Similarity report of the project report below 16% duly measured using ‘Turnitin’ similarity measure software (available at central library).

[ListParagraph] References (The listing of references should be typed 2 spaces below the heading “REFERENCES” in alphabetical order in single spacing left – justified.  It should be numbered consecutively (in square [ ] brackets, throughout the text and should be collected together in the reference list at the end of the report. The references should be numbered in the order they are used in the text. The name of the author/authors should be immediately followed by the year and other details).  Typical examples of the references are given below: 
[1]Ariponnammal, S. and Natarajan, S. (1994) ‘Transport Phonomena of SmSel – X Asx’, Pramana – Journal of Physics Vol.42, No.1, pp.421-425. 

[ListParagraph] 11.Reflection of the team members on the project 
[ListParagraph] Write what you learned as a team.
[ListParagraph]  Write what you learned as a member.
[ListParagraph]  Write a thoughtful paragraph on strengths and weaknesses of your design process.
[ListParagraph]  In addition, following points should be complied with:
[ListParagraph] Page numbering
[ListParagraph] Numbering of appendices, figures and tables and their reference in the text.
[ListParagraph] For general layout of report, any standard text book layout may be referred.
Report Specifications
[ListParagraph] Project Report’s Cover Type:  Hard-bound
[ListParagraph] Color of Project Report Cover:  Navy Blue only with white alphabets (as per Annexure 1)
[ListParagraph] Number of Copies: 2 per group (Internal/External exam + Individual or Student’s copy)
[ListParagraph] Paper Size (orientation): A4 (portrait)
[ListParagraph] Margins: 1” top / bottom / right and 1.5” left
[ListParagraph] Font Type: Times New Roman
[ListParagraph] Font Size: 16 bold for chapter names, 14 bold for headings and 12 for normal text
[ListParagraph] Line Spacing: 1.5 throughout
[ListParagraph] Page Numbering:  Bottom center of page in the format – Page 1 of N
NOTE: Project report must not contain any description of the following except only a relevant and short mention – technology or platform or OS or tools used or any language details.  It must be more focused on project work carried out and its implementation details without including any source code.
Details of CD
CD of the project work is required to be pasted on the back cover of the project report in clear packet, which should include the following folders and contents:
[ListParagraph] Presentation (should include a PPT about project in not more than 20-30 slides)
[ListParagraph] Documentation (should include a word file of the project report)
[ListParagraph] Source Code (full source code of the project with libraries used)
[ListParagraph] Program (final running copy of the project executable)
[ListParagraph] Support (any third party tools used or runtime environment setups that are required to run the project)
[ListParagraph] Help (user manual on how to run the project)
NOTE: CD must be checked for any harmful viruses before submission. Source Code and Program folders may be combined into single folder Project if it’s a web project etc.
